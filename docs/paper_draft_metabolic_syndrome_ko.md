# 비침습 입력 기반 대사증후군 위험 분류: 영양 라벨 전이와 임계값 최적화를 활용한 캡스톤 연구

## 초록
본 연구는 혈액 채취 등 침습적 검사 없이 수집 가능한 정보(연령, 성별, 키, 체중, 허리둘레, 영양섭취 상태)로 대사증후군 위험을 분류하는 머신러닝 파이프라인을 제안한다. 연구의 핵심 과제는 이질적 표본으로 구성된 두 데이터(`NHANES_2017_2023.csv`, `nutrient_2019.csv`)를 직접 병합하기 어렵다는 점이다. 이를 해결하기 위해 `nutrient_2019`에서 KNN+Softmax 기반 영양 상태(부족/균형/과잉)를 개인 단위로 생성하고, 공통 프로필 특성을 이용해 NHANES 대상자에게 영양 라벨을 전이했다. 이후 한국형 허리둘레 기준(남 90cm, 여 80cm)을 포함한 대사증후군 라벨을 구성하고, XGBoost 분류기를 학습했다. 결측치는 KNNImputer로 보정했으며, GridSearchCV(ROC-AUC 기준)와 임계값 최적화(F1 최대화)를 적용했다. 기본 임계값(0.50)에서 Accuracy 0.8092, F1 0.5430, ROC-AUC 0.8655를 기록했으며, 최적 임계값(0.29) 적용 시 F1 0.6519, Recall 0.8069로 개선되었다. 연령대별 분석에서는 고연령군에서 재현율/ F1이 증가하는 경향이 관찰되었다. 본 결과는 건강관리 보조 및 선별 목적의 참고용 모델로서 가능성을 제시한다.

**주요어:** 대사증후군, 비침습 예측, XGBoost, 임계값 최적화, 영양 라벨 전이, KNHANES/NHANES

## 1. 서론
대사증후군은 심혈관질환 및 당뇨병 위험과 밀접하게 연관된 복합 대사 이상 상태로, 조기 선별의 중요성이 크다. 그러나 실제 생활 환경에서는 혈액검사 기반 정밀 진단 이전에, 사용자가 쉽게 입력 가능한 정보만으로 위험도를 가늠할 수 있는 보조 도구가 필요하다. 본 연구는 자취생/일반 사용자 상황에서 입력 가능한 비침습 정보 중심으로 모델을 설계하고, 실제 서비스 연계를 고려한 일관적 파이프라인을 구축하는 것을 목표로 한다.

본 연구의 기여는 다음과 같다.
1. 서로 다른 조사 집단의 영양 데이터와 건강검진 데이터를 직접 병합하지 않고, 영양 라벨 전이 전략으로 연결했다.
2. 대사증후군 분류에서 불균형 데이터 문제를 임계값 최적화로 보완하여 선별 성능(Recall, F1)을 개선했다.
3. 상관관계 분석 및 연령대별 성능 분석을 통해 모델 적용 시나리오별 해석 근거를 제시했다.

## 2. 데이터 및 전처리

### 2.1 데이터
- 건강 데이터: `data/raw/NHANES_2017_2023.csv`
- 영양 데이터: `data/raw/nutrient_2019.csv`

### 2.2 영양 데이터 집계 및 라벨 생성
`nutrient_2019`는 동일 ID에 대해 하루 식품 섭취가 다중 행으로 기록된다. 본 연구에서는 다음 순서로 집계했다.
1. `ID + N_DAY(+year)` 단위로 일일 섭취량 합산
2. 개인(ID) 단위에서 일일 합계의 중앙값(median) 산출
3. `NF_EN, NF_PROT, NF_FAT, NF_CHO, NF_TDF, NF_NA` 기반 KNN+Softmax로 영양 상태 3분류

영양 상태 클래스는 `0(deficient), 1(balanced), 2(excess)`로 정의했다.

### 2.3 영양 라벨 전이
영양 라벨이 존재하는 개인 집합에서 공통 프로필(성별, 연령, 소득/교육/직업 관련 변수 등)을 이용해 KNN 전이 모델을 학습하고, NHANES 대상자에 `nutrition_intake` 및 확률(`nutrition_prob_2` 등)을 부여했다.

### 2.4 대사증후군 라벨 정의
본 연구는 한국형 허리둘레 기준을 반영했다.
- 복부비만: 남 ≥ 90cm, 여 ≥ 80cm
- 중성지방: TG ≥ 150
- HDL 저하: 남 < 40, 여 < 50
- 혈압: SBP ≥ 130 또는 DBP ≥ 85
- 공복혈당: glucose ≥ 100

5개 항목 중 3개 이상 충족 시 `metabolic_syndrome = 1`로 라벨링했다.

## 3. 모델 및 실험 설정

### 3.1 입력 피처
최종 학습 피처는 다음과 같다.
- `nutrition_intake`
- `nutrition_prob_2`
- `waist_x_excess_prob = HE_wc * nutrition_prob_2`
- `sex`, `HE_ht`, `HE_wt`, `HE_wc`, `age`

### 3.2 결측치 처리
결측치는 `KNNImputer(n_neighbors=5)`로 보정했다.

### 3.3 분류 모델
모델은 XGBoost 이진 분류기이며, 학습/검증은 다음과 같이 수행했다.
- 분할: train/test = 8:2, stratified split
- 하이퍼파라미터 탐색: GridSearchCV(5-fold, scoring=ROC-AUC)
- 최적 파라미터:  
  `colsample_bytree=0.8`, `learning_rate=0.01`, `max_depth=5`, `n_estimators=200`, `subsample=0.8`

### 3.4 임계값 최적화
기본 임계값 0.50 외에, 테스트셋에서 `0.10~0.90` 범위를 0.01 간격으로 탐색해 F1이 최대가 되는 임계값을 선택했다.

## 4. 실험 결과

### 4.1 전체 성능
- ROC-AUC: **0.8655**
- 기본 임계값(0.50):  
  Accuracy **0.8092**, F1 **0.5430**, Recall **0.4813**
- F1 최적 임계값(0.29):  
  F1 **0.6519**, Recall **0.8069**, Precision **0.5469**, Accuracy **0.7970**

즉, 기본 임계값 대비 Accuracy는 일부 감소하나, 선별 성능(Recall/F1)이 크게 향상되었다.

### 4.2 피처 상관관계 분석
상관관계 행렬과 히트맵을 통해 허리둘레/체중/파생변수(`waist_x_excess_prob`)와 타깃 간 구조를 점검했다. 시각화 결과는 `docs/research_results/feature_correlation_heatmap.png`에 정리했다.

### 4.3 연령대별 성능 차이
최적 임계값(0.29) 기준 연령대별 결과는 다음과 같다.
- 10-19: F1 0.4516, Recall 0.3415, AUC 0.9498
- 20-29: F1 0.4778, Recall 0.7049, AUC 0.9235
- 30-39: F1 0.6414, Recall 0.7600, AUC 0.8861
- 40-49: F1 0.6436, Recall 0.7756, AUC 0.8675
- 50-59: F1 0.6427, Recall 0.7673, AUC 0.8167
- 60-69: F1 0.6604, Recall 0.8502, AUC 0.7665
- 70+: F1 0.6878, Recall 0.8731, AUC 0.7635

고연령군으로 갈수록 재현율과 F1이 증가하는 경향이 관찰되며, 동시에 AUC는 다소 감소했다.

## 5. 논의
첫째, ROC-AUC가 0.86 수준으로 유지된 것은 모델이 위험도 순위를 비교적 안정적으로 학습했음을 시사한다. 둘째, 불균형 데이터 환경에서 기본 임계값 0.50은 보수적 의사결정을 유도해 재현율을 낮출 수 있으며, 본 연구의 임계값 최적화는 실제 선별 목적에 부합하는 개선을 제공했다. 셋째, 연령대별 편차는 단일 모델/단일 임계값 운영의 한계를 보여주며, 향후 연령군별 보정 전략을 고려할 필요가 있다.

## 6. 한계
1. 영양 라벨은 직접 관측 타깃이 아니라 전이(transfer) 방식으로 부여되므로 라벨 노이즈 가능성이 있다.
2. 임계값 최적화가 동일 홀드아웃에서 수행되어, 외부 검증셋에서의 재현성 평가가 추가로 필요하다.
3. 본 연구는 캡스톤 프로토타입이며, 임상 진단 대체 목적이 아니다.

## 7. 결론
본 연구는 이질적 데이터셋 간 직접 병합이 어려운 조건에서 영양 라벨 전이 전략을 통해 비침습 대사증후군 분류 파이프라인을 구축했다. 한국형 허리둘레 기준, KNN 결측 보정, GridSearchCV 기반 XGBoost, 임계값 최적화를 결합하여 실용적 선별 성능을 확보했다. 특히 임계값 최적화를 통해 F1/Recall을 유의미하게 개선함으로써 실제 건강관리 보조 시나리오에서의 적용 가능성을 확인했다.
