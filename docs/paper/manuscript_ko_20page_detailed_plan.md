## 2.3 방법론 (약 5p)
## 2.3.1 데이터 기술
- 건강 데이터: `data/raw/NHANES_2017_2023.csv`
- 영양 데이터: `data/raw/nutrient_2019.csv`
- 표본 분할: train `38174`, test `9544`

### 상세 기술 포인트
- 각 데이터의 수집 단위(개인 vs 일일 다중 식사 레코드)
- 변수 스케일 차이(신체계측 vs 영양량)
- 병합 제약(공통 개인 식별 불가/코호트 상이)

## 2.3.2 라벨 전이 설계
- 1단계: 영양 데이터에서 개인 단위 상태 산출
- 2단계: 공통 프로필 기반 KNN 전이로 NHANES에 영양 라벨/확률 부여
- 핵심 파생변수: `nutrition_prob_2`, `waist_x_excess_prob`

### 수식(논문 본문 삽입 권장)
`waist_x_excess_prob = HE_wc * nutrition_prob_2`

## 2.3.3 생활패턴 스무딩 피처
- 평균/표준편차/IQR: `pattern_energy_ratio_mean/std/iqr`
- 상태 비율: `pattern_excess/deficient/balanced_day_ratio`
- 밀도형: `pattern_protein/fat/carb/sodium_density_mean`

### 서술 포인트
- 단일 일자 노이즈를 개인 분포 통계로 완화
- “섭취량 절대치” 대신 “상태 분포”를 학습 신호로 제공

## 2.3.4 타깃 정의(대사증후군)
- 복부비만(남 90, 여 80), TG, HDL, 혈압, 공복혈당
- 5개 중 3개 이상 충족 시 양성

### 명확화 문장(권장)
“본 연구의 타깃은 임상 진단 확정값이 아닌, 선별 목적의 규칙 기반 위험 라벨이다.”

## 2.3.5 모델/학습/평가
- 모델: XGBoost
- 결측치: KNNImputer
- 튜닝: GridSearchCV(5-fold, ROC-AUC)
- 최적 파라미터:
  - `colsample_bytree=0.8`
  - `learning_rate=0.01`
  - `max_depth=5`
  - `n_estimators=200`
  - `subsample=0.8`

## 2.3.6 임계값 최적화 정책
- 후보: `0.10~0.90` (0.01 step)
- 정책 A: 기본(0.50)
- 정책 B: F1 최대
- 정책 C: Recall 우선(단, precision >= 0.50)

---

## 2.4 결과 (약 4p)
## 2.4.1 기본 성능표(필수 표 1)
- Default(0.50): Accuracy `0.8109`, Precision `0.6319`, Recall `0.4720`, F1 `0.5404`, ROC-AUC `0.8664`
- Best-F1(0.36): Accuracy `0.8105`, Precision `0.5740`, Recall `0.7571`, F1 `0.6530`
- Recall-priority(0.23): Precision `0.5052`, Recall `0.8603`, F1 `0.6366`

## 2.4.2 A/B 비교(필수 표 2)
- Full(18 features) vs Light(11 features)
- Full이 F1/AUC에서 근소 우세

## 2.4.3 임계값-지표 곡선(필수 그림)
- `xgboost_threshold_metrics.csv`로 threshold vs (precision/recall/f1/accuracy) 그래프
- 본문 메시지:
  - 임계값 하향 시 Recall↑, Precision↓
  - 선별 목적에서는 0.36 또는 0.23이 운영상 유리

## 2.4.4 연령대별 성능(필수 표 3 + 그림)
- 고연령군에서 Recall/F1 증가 경향
- AUC는 감소 경향
- 해석: 군별 위험도 분포와 의사결정 임계값 민감도 차이

## 2.5 논의 (약 4p)
## 2.5.1 핵심 해석
- AUC 0.86대 유지: 전반적 순위화 능력 확보
- threshold 조정 효과가 크며, 실제 서비스 목적(미탐 최소화)에 부합
- 스무딩 피처의 성능 기여는 “크게 폭발적”이기보다 “일관된 소폭 개선”

## 2.5.2 운영 관점 제안
- 기본 모드: F1 최적(0.36)
- 고위험 미탐 최소화 모드: Recall 우선(0.23)
- 앱 UX 측면:
  - 결과는 “진단”이 아니라 “위험 신호”로 표기
  - 사용자 액션(검진 권고, 식습관 개선 가이드)과 연결
